# DCQ-GNN: ConvexConcave Quadratic Spectral Filtering for Graph Neural Networks

A PyTorch implementation of [DCQ-GNN](https://arxiv.org/abs/2606.24956), a graph neural network that uses second-order (quadratic) spectral filters with learned convex/concave curvature control and node-adaptive gated fusion.

## Requirements

- Python 3.8+
- PyTorch 2.0+
- PyTorch Geometric
- Optuna
- PyYAML
- NumPy
- DeepRobust (optional, for Metaattack evaluation)

Install dependencies:

```bash
pip install torch torch_geometric optuna pyyaml numpy deeprobust
```

## Project Structure

```
├── main.py          # Entry point: training and hyperparameter search
├── model.py         # DCQ_GNN model and QuadraticFilterLayer definitions
├── engine.py        # Training, validation, testing, and Optuna objective
├── dataset.py       # Dataset loading with optional adversarial attacks
├── utils.py         # Config loading, attacks (DICE, Metaattack, Noise)
├── config/          # YAML config files (auto-generated by search mode)
├── run.sh           # Batch training script
└── search.sh        # Batch hyperparameter search script
```

## Usage

### Step 1: Hyperparameter Search (Tuning)

Find optimal hyperparameters for dataset using Optuna:

```bash
# Single dataset
python main.py Cora -m search --epochs 500

# Multiple datasets (see search.sh)
python main.py Photo -m search
python main.py PubMed -m search
```

Search results are saved to `./config/<dataset>.yaml`. Key hyperparameters tuned:

| Parameter | Range | Description |
|-----------|-------|-------------|
| `num layer` | 1-3 | Number of QuadraticFilterLayers |
| `dr` | 0.0-0.9 | Dropout rate |
| `lr` | 1e-4 - 1e-2 | Learning rate |
| `wd` | 1e-6 - 1e-2 | Weight decay |
| `feature transform` | True/False | NormalizeFeatures transform |
| `low-pass opt {layer}` | cvx/ccv | Low-pass convex or concave |
| `high-pass opt {layer}` | cvx/ccv | High-pass convex or concave |

### Step 2: Training

Train with the discovered (or provided) configuration:

```bash
# Basic training (10 runs with averaged results)
python main.py Cora -m train

# With custom settings
python main.py Cora -m train -n 10 --epochs 1000 --device 0

# With adversarial attack (DICE, Metaattack, or Noise)
python main.py Cora -m train --atk DICE --ptb_rate 0.1

# Using a specific config file
python main.py Cora -m train --config_file Cora.yaml
```

**Command-line arguments:**

| Argument | Default | Description |
|----------|---------|-------------|
| `dataset` | (required) | Dataset name |
| `-m, --mode` | train | `train` or `search` |
| `-n, --num_run` | 10 | Number of training runs to average |
| `--config_file` | None | Path to YAML config file |
| `--train_ratio` | 0.6 | Fraction of nodes for training |
| `--hidden_features` | 64 | Hidden layer dimension |
| `--epochs` | 1000 | Maximum training epochs |
| `--atk` | None | Attack type: DICE, Metaattack, Noise |
| `--ptb_rate` | 0.1 | Perturbation ratio for attacks |
| `--device` | 0 | GPU device ID |

### Available Datasets

- **Homophilic:** Cora, CiteSeer, PubMed, CS, Physics, Computers, Photo, WikiCS
- **Heterophilic:** chameleon, crocodile, Squirrel, Actor, Cornell, Texas, Wisconsin, Roman-empire, Amazon-ratings, Minesweeper, Tolokers, Questions
